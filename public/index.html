How to Use a VPS with SSH Access as a Git Repository

Hosting your own Git repositories is a great way to control your code and collaborate with others without relying on third-party services. If you have a VPS (Virtual Private Server) with SSH access, you can turn it into a private Git server in just a few steps. This guide will walk you through the process.

Prerequisites

  • A VPS running Linux (e.g., Ubuntu, Debian, CentOS)
  • SSH access to the VPS
  • Git installed on both your local machine and the VPS

Step 1: Install Git on Your VPS

First, connect to your VPS using SSH:

ssh username@your_vps_ip

Then, install Git:

Ubuntu/Debian:

sudo apt update && sudo apt install git

CentOS/RHEL:

sudo yum install git

Step 2: Create a Git User (Optional but Recommended)

It’s a good security practice to create a dedicated user for managing Git repositories.

sudo adduser git

You can now log in as this user:

su - git

Step 3: Create a Bare Git Repository

A bare repository is used for sharing and collaboration. It does not have a working directory and is suitable for pushing and pulling.

mkdir -p ~/repos/project.git
cd ~/repos/project.git
git init --bare

Step 4: Configure SSH Access (Optional)

If you set up a git user, add your local SSH public key to ~git/.ssh/authorized_keys on the VPS to allow passwordless authentication:

  1. On your local machine, display your public key: sh cat ~/.ssh/id_rsa.pub
  2. Copy the output, then on the VPS (as git user): sh mkdir -p ~/.ssh echo "your_public_key_here" >> ~/.ssh/authorized_keys chmod 700 ~/.ssh chmod 600 ~/.ssh/authorized_keys

Step 5: Push Your Code to the VPS Git Repository

On your local machine, add the VPS as a remote:

git remote add origin git@your_vps_ip:~/repos/project.git

Or, if you’re using a different username:

git remote add origin username@your_vps_ip:~/repos/project.git

Now push your code:

git push -u origin master

Step 6: Clone from the VPS Repository

To clone the repository from another machine:

git clone git@your_vps_ip:~/repos/project.git

Tips and Best Practices

  • Backups: Regularly back up your VPS and repositories.
  • SSH Security: Use SSH keys, disable password authentication, and consider changing the default SSH port.
  • Collaboration: Add collaborators’ public keys to authorized_keys.
  • Repository Management: Use directory structure and permissions to organize and protect your repos.

Conclusion

By following these steps, you can use your VPS as a private Git server with SSH access. This setup gives you full control and privacy over your code repositories, making it ideal for personal projects or small teams.
