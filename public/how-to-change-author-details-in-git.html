How to Change Author Details in Git

Whether you're correcting a past mistake or updating your credentials after switching email addresses, modifying author details in Git can be essential for maintaining clean, accurate commit history. In this guide, we’ll walk you through how to change the author name and email for future commits as well as for past commits.

Change Author Details for Future Commits

To change the name and email Git uses for your commits going forward, use the following commands: 


git config --global user.name "Your New Name"
git config --global user.email "your.email@example.com"

Notes:

  • Use the --global flag to apply these settings globally across all repositories on your machine.
  • If you want to set these details for a specific repository only, navigate to that repository and run the commands without the --global flag.

git config user.name "Your New Name"
git config user.email "your.email@example.com"

Change Author Details for Past Commits (Interactive)

If you need to rewrite the author information in past commits, you can use an interactive rebase (for recent commits): 


git rebase -i HEAD~n

Replace n with the number of commits you want to change. When the editor opens:

  1. Change pick to edit next to the commits you want to modify.
  2. For each commit marked for editing, run:

git commit --amend --author="New Name <new.email@example.com>"
  1. Then continue the rebase with:

git rebase --continue

Verifying Changes

To verify what Git is currently using as your name and email: 


git config user.name
git config user.email

To check what’s in the commits: 


git log --pretty=format:"%h %an <%ae> - %s"
